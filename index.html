Client-Side Security Tester

Browser-based Security Analysis & Detection Platform

Live • Deployed Intermediate Security Application

A browser-based educational security analysis platform that performs heuristic detection against user-supplied input across eight independent analyzers. Produces structured findings with severity scoring, remediation guidance, and exportable security reports — all running entirely client-side with no backend or build tools required.

Security Topics Covered:

XSS DOM Sinks URL Reputation JWT Security CSP Analysis HTTP Headers Cookie Security Storage Security

Technical Skills Demonstrated:

ES Modules DOM APIs Regex Detection State Management CSS Grid & Flexbox Modular Architecture Dynamic UI Rendering Input Sanitization

Tech Stack:

HTML CSS JavaScript
🔗 Live 💻 Code
  • XSS detection — script tags, event handlers, javascript: URIs, and iframe injection
  • DOM sink analysis — flags innerHTML, eval(), document.write(), location.href, and insertAdjacentHTML
  • URL reputation — detects raw IP domains, excessive subdomains, brand impersonation, homograph characters, and encoded hostnames
  • JWT analyzer — decodes tokens and checks for alg:none, missing expiry, and sensitive payload data
  • CSP analyzer — flags unsafe-inline, unsafe-eval, wildcard sources, and missing frame-ancestors
  • Security header analysis — checks HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, and Permissions-Policy
  • Cookie security — per-cookie checks for missing HttpOnly, Secure, SameSite, and SameSite=None without Secure
  • Storage security — scans JavaScript code for sensitive keys, JWT values, Bearer tokens, and eval() with storage data
  • Modular ES Module architecture — each analyzer is a pure function returning a typed findings array
  • Centralised findings store with severity-based risk scoring (High +40, Medium +20, Low +10)
  • Live severity bar chart on every scan — no canvas, no library, pure CSS transition
  • Findings filter bar (All / High / Medium / Low) with per-severity colour states
  • Activity history with instant keyword search and match highlighting
  • JSON export and formatted HTML security report generation
  • Full documentation page with sticky sidebar TOC, copy-enabled code blocks, and scroll-tracked active links
  • Problem: Adding new tools required touching dashboard.js to clear inputs on reset
  • Solution: Reset callback registry — each tool registers its own clear function so the dashboard never knows input IDs

🔐 Security Breakdown

  • Input Safety: All user input passes through escapeHTML() before any DOM insertion — no raw rendering anywhere in the codebase.
  • No Data Exfiltration: All analysis runs entirely in the browser — no network requests, no external APIs, no data ever leaves the page.
  • Scope Isolation: ES Module imports prevent global scope pollution and accidental cross-module state leaks.
  • Storage: The tool itself uses no localStorage or sessionStorage — ironic given it detects misuse of both.
  • External Links: All outbound links use rel="noopener noreferrer" to prevent tab-napping.

Detection logic is heuristic and educational — designed to demonstrate vulnerability patterns and detection techniques, not replace professional security tooling.
