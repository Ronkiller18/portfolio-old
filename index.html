Client-Side Security Tester

Browser-based security analysis platform with eight detection modules

Live • Deployed Intermediate Frontend

Role: Designed and developed the full frontend architecture — modular analyzer system, multi-tool dashboard, findings pipeline, and documentation page.

Tech Stack:

HTML CSS Vanilla JavaScript ES Modules
🔗 Live 💻 Code

    Detection Modules

    XSS: Detects script tags, inline event handlers, javascript: URIs, and iframe injection patterns.

    DOM Sinks: Flags unsafe APIs — innerHTML, eval(), document.write(), location.href assignment, and insertAdjacentHTML.

    URL Reputation: Identifies raw IP domains, excessive subdomain depth, brand impersonation, homograph characters, and URL-encoded hostnames.

    JWT: Decodes tokens and checks for alg:none, missing expiration claims, and sensitive data in the payload.

    CSP: Parses Content Security Policy headers and flags unsafe-inline, unsafe-eval, wildcard sources, and missing frame-ancestors.

    Security Headers: Checks HTTP response headers for missing or misconfigured HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, and Permissions-Policy.

    Cookie Security: Analyses Set-Cookie headers per cookie — detects missing HttpOnly, Secure, SameSite, and SameSite=None without Secure.

    Storage Security: Scans JavaScript code for sensitive data written to localStorage or sessionStorage, JWT values, Bearer tokens, and eval() with storage data.

  • Modular ES Module architecture — each analyzer is a pure function returning a typed findings array
  • Centralised findings store with severity-based risk scoring (High +40, Medium +20, Low +10)
  • Live severity bar chart updates on every scan — no canvas, no library, pure CSS animation
  • Findings filter bar (All / High / Medium / Low) with per-severity colour states
  • Activity history with instant keyword search and match highlighting
  • JSON export and formatted HTML security report generation
  • Full documentation page with sticky TOC, copy-enabled code blocks, and scroll-based active link tracking
  • Problem: Needed a clean, extensible way to add new security tools without touching existing code
  • Solution: Reset callback registry in dashboard.js — each tool registers its own clear function, so the dashboard never needs to know tool input IDs

🔐 Security Notes

  • All user input rendered through escapeHTML() — no raw DOM insertion anywhere
  • No data leaves the browser — all analysis runs client-side with no network requests
  • No localStorage or sessionStorage used by the tool itself
  • ES Module imports prevent global scope pollution
  • External links use rel="noopener noreferrer"

Detection logic is heuristic and educational — designed to demonstrate vulnerability patterns, not replace professional security tooling.
