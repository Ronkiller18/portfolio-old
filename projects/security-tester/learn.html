Project Overview

Client-Side Security Tester is a browser-based educational security analysis tool designed to demonstrate common client-side security risks.

The project performs heuristic analysis against user-supplied input and highlights potentially dangerous patterns commonly associated with web application attacks.

Supported Modules

  • XSS Detection
  • DOM Sink Analysis
  • Phishing URL Detection
  • JWT Inspection
  • Content Security Policy Review

Cross-Site Scripting (XSS)

Description

Cross-Site Scripting occurs when untrusted input is interpreted as executable JavaScript by a browser.

Why It Matters

Successful XSS attacks can lead to session theft, account compromise, credential harvesting, and malicious actions performed on behalf of users.

Example Payloads

                <script>alert(1)</script>

                <img src=x onerror=alert(1)>

                <svg onload=alert(1)>

Mitigation

  • Validate user input
  • Escape HTML output
  • Use Content Security Policy
  • Avoid rendering raw user content

DOM-Based XSS

Description

DOM XSS occurs when JavaScript reads attacker-controlled input and inserts it into dangerous browser APIs.

Why It Matters

Vulnerable DOM sinks may allow attackers to execute arbitrary JavaScript without interacting with a backend server.

Common Dangerous Sinks

                element.innerHTML = userInput

                document.write(userInput)

                eval(userInput)

                insertAdjacentHTML(...)

Mitigation

  • Prefer textContent
  • Avoid eval()
  • Sanitize user-controlled data
  • Use secure templating systems

Phishing URL Detection

Description

Phishing attacks commonly use deceptive URLs designed to mimic legitimate login pages and trusted services.

Detection Logic

The analyzer searches for suspicious keywords and risky domain patterns often associated with phishing campaigns.

Example

                secure-login-update.xyz

                verify-account-now.tk

                bank-account-security.ml

Mitigation

  • Verify domain ownership
  • Inspect URLs carefully
  • Enable MFA when possible
  • Never trust login links blindly

JWT Security Analysis

Description

JSON Web Tokens are commonly used for authentication and authorization in modern web applications.

Detected Issues

  • alg:none usage
  • Missing expiration claims
  • Sensitive data exposure

Example Token

                eyJhbGciOiJub25lIiwidHlwIjoiSldUIn0.
                eyJ1c2VyIjoiYWRtaW4iLCJwYXNzd29yZCI6InNlY3JldCJ9.

Mitigation

  • Always require signatures
  • Use exp claims
  • Avoid secrets in payloads
  • Rotate signing keys regularly

Content Security Policy (CSP)

Description

CSP is a browser security mechanism used to restrict the sources from which content may be loaded and executed.

Detected Issues

  • unsafe-inline
  • unsafe-eval
  • Wildcard script sources
  • Missing frame-ancestors

Example Weak Policy

                default-src 'self';
                script-src 'unsafe-inline' *;
                object-src *;

Mitigation

  • Use nonces or hashes
  • Avoid unsafe-inline
  • Avoid unsafe-eval
  • Define frame-ancestors

Risk Scoring Model

The dashboard generates a simplified risk score based on detected findings.

  • High Severity = 40 Points
  • Medium Severity = 20 Points
  • Low Severity = 10 Points

This scoring model is intentionally simplified for educational purposes and should not replace professional risk assessment methodologies.
