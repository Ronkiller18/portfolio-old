🔐 Authentication Vulnerability Lab

This interactive lab demonstrates common authentication and access control vulnerabilities found in web applications.

  • 🔴 Weak client-side authentication
  • 🟠 Insecure session storage (localStorage)
  • 🔴 Broken access control (direct page access)

Try to bypass the login and access restricted functionality using browser tools.

🔐 Login (Vulnerable)

🧠 Vulnerability Breakdown

🔴 Weak Authentication (Client-Side)

Description:
The login logic is implemented entirely in client-side JavaScript, making it visible and modifiable by users.

Vulnerability:
Credentials are hardcoded in the browser and validated locally.

Impact:
Attackers can bypass authentication by modifying JavaScript or directly invoking functions.

Example Attack:


        login = function() {
        document.getElementById("message").textContent = "Login successful";
        }

Fix:
Authentication should be handled securely on a backend server.

🟠 Insecure Session Storage

Description:
The application stores authentication state in localStorage, which is fully controlled by the user.

Vulnerability:
A session flag (isLoggedIn) is stored in the browser after login. This value can be manually modified by an attacker.

Impact:
Attackers can bypass authentication by setting the session value directly, gaining unauthorized access.

Example Attack:


        localStorage.setItem("isLoggedIn", "true");

Fix:
Session management should be handled securely using server-side sessions or HTTP-only cookies.
